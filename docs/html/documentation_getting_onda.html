Accessing OnDA at the LCLS and Petra III Facilities¶

OnDA comes pre-installed at some beamlines of the Petra III and LCLS facilities:

  • LCLS:
    • AMO
    • CXI
  • Petra III
    • P11

Beamline scientists can provide information on how to access and use OnDA. However, for reference, instructions can also be found in the following paragraphs.

LCLS - AMO and CXI¶

In order to run OnDA at these beamlines, access to the monitoring nodes is needed. They are the only machines which can access the real-time data stream. Once access has been granted, the latest version of OnDA can be reached by activating an Anaconda python environment:

# On one of the monitoring nodes, for example, daq-cxi-mon07
source /reg/g/cfel/onda/onda.sh

Petra III - P11¶

OnDA is available at this beamline on the ‘eval01’ machine and must be run on it. Once logged in, the following command can be used to activate the Anaconda python environment in which OnDA is installed:

# On eval01.desy.de
source /home/p11user/CfelSoft/onda/onda.sh

Installing OnDA on a Local Machine¶

OnDA can easily be installed on a local machine from the Python Package Index. It is also available as a package for the Anaconda Python distribution.

Installation from the Python Package Index¶

OnDA can be installed from the Python Package Index (PyPI) using the ‘pip’ command. This is the preferred installation method. The only requirements are:

  • An actively supported version of Python (2.7, 3.5, 3.6, or 3.7).
  • The ‘pip’ utility for the corresponding version of python.

Python 3:

It is recommended to install OnDA in a clean Python virtual environment. The environment can be created using the venv module from the Python 3 Standard Library:

python3 -m venv onda_env

The environment can be activated by sourcing a file:

source onda_env/bin/activate

Finally, OnDA can be installed using the ‘pip’ command. In order to install only the monitoring part of OnDA (without the graphical interfaces):

python3 -m pip install onda

In order to install OnDA together with the Qt5-based graphical interfaces:

python3 -m pip install onda[GUI]

Python 2:

For Python 2, the process, is very similar, except that the virtualenv tool must be used to create the enviroment:

python2 -m virtualenv onda_env

The rest of the process is then identical:

source onda_env/bin/activate

Then:

python2 -m pip install onda

Or:

python2 -m pip install onda[GUI]

Installation in Anaconda¶

Python 3:

Packages for OnDA and its dependencies are available from the ‘ondateam’ channel of https://anaconda.org. It is reccomended to install onda in a clean Python enviroment, which can be created using the following command:

conda create -n onda_env python=3

The enviroment can then be activated using the following command:

conda activate onda_env

Finally, OnDA can be installed using the ‘conda install’ command:

conda install --channel conda-forge --channel ondateam onda

Python 2:

For Python 2 the process is very similar:

conda create -n onda_env python=2

Then:

conda activate onda_env

And:

conda install --channel conda-forge --channel ondateam onda
