The data_handlers_psana Module¶

Retrieval and handling of data events from psana.

This module contains DataEventHandlers for events retrieved from the psana framework.

class LclsBaseDataEventHandler(monitor_parameters, source)[source]¶

Bases: om.data_retrieval_layer.base.OmDataEventHandler

DataEventHandler for events recovered from psana at LCLS.

See documentation of the corresponding function in the base class. This class generically handles events retrieved from the psana framework at the LCLS facility. It should be subclassed to work with specific detectors.

Parameters

  • monitor_parameters (MonitorParams) – An object storing the OM monitor parameters from the configuration file.

  • source (str) – A string describing the data source.

initialize_event_handling_on_collecting_node(node_rank, node_pool_size)[source]¶

Initializes psana event handling on the collecting node.

See documentation of the corresponding function in the base class. The psana data source does not need to be initialized on the collecting node, therefore this function does nothing.

Parameters

  • node_rank (int) – The rank, in the OM pool, of the processing node calling the function.

  • node_pool_size (int) – The total number of nodes in the OM pool, including all the processing nodes and the collecting node.

Return type

Any

An optional initialization token.

event_generator(node_rank, node_pool_size)[source]¶

Retrieves psana events.

See documentation of the corresponding function in the base class. Each psana event contains data related, timewise, to a single detector frame.

Parameters

  • node_rank (int) – The rank, in the OM pool, of the processing node calling the function.

  • node_pool_size (int) – The total number of nodes in the OM pool, including all the processing nodes and the collecting node.

Yields

A dictionary storing the event data.
